# AECODE Revenue & Enrollment OS

## Scope

Dashboard publico y sanitizado construido desde la pestana `BBDD Ventas General`.

## Rules

- No publicar nombres, correos, telefonos, links de vouchers ni comentarios con datos personales.
- Mantener solo metricas agregadas, usuarios anonimos y registros anonimizados.
- Si se necesita detalle nominal, debe vivir en Google Sheets o un entorno autenticado, no en GitHub Pages.
- El dashboard debe responder decisiones comerciales: ventas, inscritos, usuarios, productos, medios de pago, pendientes, anulados y calidad de datos.
- La app debe poder desplegarse como sitio estatico sin backend.
